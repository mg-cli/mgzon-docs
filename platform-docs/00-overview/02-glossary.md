# Glossary of Terms

## A

| Term | Definition |
|------|------------|
| **Admin** | Platform administrator with full system access |
| **AML** | Anti-Money Laundering - compliance checks for financial transactions |
| **API Key** | Authentication token for API access |
| **App** | Application or integration published on the platform |
| **Approval Workflow** | Process for reviewing and approving content before publication |
| **ATM Code** | One-time code for ATM withdrawals from MGPay account |
| **Audit Log** | Record of all administrative actions for compliance |

## B

| Term | Definition |
|------|------------|
| **Bank Account** | User's external bank account linked for withdrawals |
| **Bid** | Offer from a freelancer to complete a project |
| **Bulk Operation** | Action performed on multiple items simultaneously |
| **Business Name** | Legal name of seller's business |

## C

| Term | Definition |
|------|------------|
| **Cart** | Temporary storage of items before purchase |
| **Category** | Classification for products and services |
| **Chargeback** | Reversal of a payment due to dispute |
| **Client** | User who posts projects and hires freelancers |
| **Commission** | Platform fee deducted from transactions |
| **Contract** | Legal agreement between client and freelancer |
| **Coupon** | Discount code for purchases |
| **Custom Section** | User-defined content section in store design |
| **Custom Site** | Customized store appearance and layout |

## D

| Term | Definition |
|------|------------|
| **Deposit** | Adding funds to MGPay account |
| **Developer** | User who creates and sells apps/integrations |
| **Dispute** | Conflict resolution process between parties |

## E

| Term | Definition |
|------|------------|
| **Employee** | Internal staff member with assigned role |
| **Escrow** | Funds held in trust during project execution |

## F

| Term | Definition |
|------|------------|
| **Fee** | Charge for service (processing, platform, withdrawal) |
| **Fulfillment** | Process of delivering ordered products |

## G

| Term | Definition |
|------|------------|
| **Gateway** | Payment processing service provider |
| **GDPR** | General Data Protection Regulation |

## H

| Term | Definition |
|------|------------|
| **Heir** | Individual entitled to inherit land |
| **Hold Period** | Time funds are held before withdrawal availability |

## I

| Term | Definition |
|------|------------|
| **Integration** | Third-party service connected to the platform |
| **Inheritance** | Distribution of land assets to heirs |
| **Invoice** | Billing document for services |

## J

| Term | Definition |
|------|------------|
| **Job** | Employment position posting |
| **Job Application** | Candidate submission for a job |

## K

| Term | Definition |
|------|------------|
| **KYC** | Know Your Customer - identity verification |

## L

| Term | Definition |
|------|------------|
| **Land** | Real estate property asset |
| **Land Owner** | Individual with legal land ownership |
| **Layout** | Visual arrangement of store sections |
| **Limit** | Maximum amount or count (daily, monthly, per transaction) |

## M

| Term | Definition |
|------|------------|
| **Marketplace** | Platform for buying and selling |
| **Metadata** | Additional data attached to records |
| **MGPay** | Main payment and wallet system |
| **Milestone** | Project phase with associated payment |
| **Mutation** | GraphQL operation that modifies data |

## N

| Term | Definition |
|------|------------|
| **Net Amount** | Transaction amount after fees |
| **Notification** | System message to user |

## O

| Term | Definition |
|------|------------|
| **Order** | Customer purchase record |
| **Owner** | Creator of a theme, template, or app |

## P

| Term | Definition |
|------|------------|
| **Page** | Store content page |
| **Payout** | Transfer of funds from platform to user |
| **Payment Method** | Means of payment (card, bank, etc.) |
| **Permission** | Specific action user is allowed to perform |
| **Plan** | Subscription plan for sellers |
| **Points** | Loyalty points earned and redeemable |
| **Position** | Employee's job role |
| **Preview** | Draft view of changes before publishing |
| **Product** | Item for sale in store |
| **Profile** | User's public information |
| **Project** | Work opportunity posted by client |
| **Public** | Visible to all users |

## Q

| Term | Definition |
|------|------------|
| **Query** | GraphQL operation that retrieves data |
| **Quick View** | Modal popup for product preview |

## R

| Term | Definition |
|------|------------|
| **Rate Limit** | Maximum API requests per time period |
| **Referral** | User recruitment with reward |
| **Refund** | Reversal of payment |
| **Report** | Generated data summary |
| **Role** | User classification with permissions |

## S

| Term | Definition |
|------|------------|
| **Salary** | Employee compensation |
| **Sandbox** | Testing environment |
| **Section** | Component of store page layout |
| **Seller** | User who sells products in store |
| **Session** | User login session |
| **Shipment** | Physical delivery of products |
| **SKU** | Stock Keeping Unit identifier |
| **Slider** | Carousel component for images |
| **Snapshot** | Saved state of preview for rollback |
| **Store** | Seller's online shop |
| **Subscription** | Recurring payment plan |

## T

| Term | Definition |
|------|------------|
| **Template** | Reusable page layout design |
| **Testimonial** | User review of product or service |
| **Theme** | Visual design system for stores |
| **Tier** | Level in developer reward system |
| **Tracking** | Order shipment tracking information |
| **Transaction** | Financial operation record |
| **Transfer** | Moving funds between users |
| **Two-Factor Auth (2FA)** | Second authentication factor |

## U

| Term | Definition |
|------|------------|
| **User** | Registered platform account |

## V

| Term | Definition |
|------|------------|
| **Variant** | Product variation (color, size) |
| **Verification** | Identity or document validation |

## W

| Term | Definition |
|------|------------|
| **Wallet** | User's fund balance container |
| **Warehouse** | Physical inventory storage location |
| **Webhook** | HTTP callback for external services |
| **Withdrawal** | Request to transfer funds out of platform |
